# Learning M&A — Workflow Visualizer

An interactive, browser-based visualization of the M&A deal lifecycle, designed
so anyone new to an M&A project can see the whole process from initial contact
through completion and post-closing.

This folder is a **design + documentation bundle** prepared in a sandboxed
session. The actual code will be built in the dedicated repo
[`learner-agent-sudo/learning-MnA`](https://github.com/learner-agent-sudo/learning-MnA).
See `TRANSFER.md` for how to copy this bundle there.

---

## Goal

Help an M&A newcomer (junior lawyer, business deal-team member, student) build
an accurate mental model of how an M&A deal flows. Each stage shows:

- What happens, in plain English (with a "practitioner depth" toggle for more)
- The key documents produced or exchanged
- Real example clauses, sourced from the [CUAD dataset](https://huggingface.co/datasets/theatticusproject/cuad-qa) where possible

## Scope of v1

- **Deal type:** Asset Purchase Agreement (APA) only
- **Branching:** A visual branch point at the LOI stage shows that the path
  diverges by deal type. The Share Purchase (SPA) and Merger paths are shown
  but greyed out / "coming soon".
- **Audience:** Mixed; a Beginner ↔ Practitioner toggle controls the depth of
  text shown in side panels.
- **Data backbone:** CUAD (510 commercial contracts, 41 lawyer-labeled clause
  categories), augmented with hand-authored short example clauses for
  APA-specific concepts CUAD does not cover (purchase price adjustments,
  disclosure schedules, MAC clauses, escrow holdback, R&W insurance, etc.).

## Out of scope for v1 (deferred to v2)

- Share Purchase (SPA) and Merger paths
- Role swimlanes (buyer / seller / lawyers / regulators)
- Indicative timelines per stage
- Due Diligence sub-stream expansion (legal / financial / tax / IP / employment)
- Mining unlabeled corpora (e.g. `albertvillanova/legal_contracts`) for real APAs

## Tech stack (proposed)

- **Framework:** Next.js 15 + React 19 + TypeScript (matches the user's existing
  ecosystem; deploys easily to Vercel / GitHub Pages with a static export)
- **Diagram:** [Mermaid](https://mermaid.js.org/) for the flow chart
  (alternative: [React Flow](https://reactflow.dev/) if we need richer
  node interactions later)
- **Styling:** Tailwind CSS (no heavy UI library)
- **Data:** Static `clauses.json` shipped with the app, generated by a one-off
  Python script (`scripts/extract_cuad.py`) that pulls CUAD from Hugging Face
  and emits ~2-5 representative clause excerpts per relevant category
- **No backend.** Fully static; can be hosted on Vercel, Netlify, or GitHub Pages.

## Why CUAD and not a "more M&A-specific" dataset

I searched the Hugging Face Hub for M&A-specific datasets (mergers,
acquisitions, asset purchase agreements, SEC EDGAR filings) and found nothing
clearly better than CUAD for our use case. Other "legal contract" datasets are
either:

- Tiny / synthetic / gated, OR
- Large but unlabeled (e.g. `albertvillanova/legal_contracts`, 106K raw
  contract texts in a single column — no annotations, would require building
  our own classifier to find M&A-relevant clauses)

CUAD is general commercial contracts, **not** purpose-built for M&A. But its
41 lawyer-labeled clause categories cover most of the clause *types* that also
appear in APAs and ancillary M&A documents. We use CUAD as a clause-pattern
library and augment with hand-authored APA-specific items where it is silent.
The UI explicitly discloses this so users are not misled.

## Decisions log (so a future session has the context)

| # | Decision | Why |
|---|---|---|
| 1 | Audience: mixed with depth toggle | User wanted both newcomers and practitioners served |
| 2 | CUAD as core data source | User specified CUAD; HF search found no clearly better M&A dataset |
| 3 | Augment CUAD with hand-authored APA gap-fills | CUAD is general commercial; APA-specific concepts (MAC, escrow, etc.) need filling in |
| 4 | Start with APA only | User wanted a single tractable deal type for v1 |
| 5 | Show SPA/Merger branch greyed-out at LOI | Sets up clean extension to v2 without re-architecting |
| 6 | DD as a single node in v1, expandable in v2 | Avoid visual clutter on first pass |
| 7 | Stage 1 (Strategy) as "pre-process" (dashed/muted) | Useful context but not strictly part of the legal workflow |
| 8 | Tech: Next.js + React + TS + Tailwind + Mermaid | Matches user's existing stack; static deploy |
| 9 | Pre-process CUAD into a static JSON, no backend | Smallest deployment surface; clauses < 1MB |
